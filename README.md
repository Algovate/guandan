# 掼蛋扑克游戏

一个界面精美精致、用户交互友好的掼蛋扑克Web游戏。

## 功能特性

- 🎮 **完整的游戏规则** - 实现完整的掼蛋游戏逻辑，支持所有牌型和规则
- 🤖 **智能AI对手** - 多种性格风格（激进型、保守型、配合型、均衡型），每局随机配置，让游戏充满变数
- 🎨 **精美UI设计** - 现代化设计语言，优雅的配色和精致的视觉效果
- ✨ **流畅动画** - 使用Framer Motion实现丝滑的交互动画
- 📱 **响应式布局** - 完美适配桌面、平板和移动设备
- 🎯 **智能提示** - 实时高亮可出牌型，一键提示最佳出牌
- 📖 **新手友好** - 详细的新手教程和游戏规则说明
- 🔔 **即时反馈** - Toast通知系统，清晰的操作反馈
- 🎴 **精美卡牌** - 精心设计的牌面，支持选中、高亮等交互状态

## 技术栈

- **React 18** + **TypeScript** - 现代化前端框架
- **Vite** - 极速构建工具
- **Tailwind CSS** - 实用优先的CSS框架
- **Framer Motion** - 强大的动画库
- **Zustand** - 轻量级状态管理
- **Inter & Poppins** - 优雅的字体系统

## UI/UX 设计亮点

- **精致的设计系统** - 统一的配色、阴影、动画规范
- **玻璃态效果** - 现代化的毛玻璃UI元素

- **响应式设计** - 完美适配各种屏幕尺寸

## 部署

### GitHub Pages

项目已配置 GitHub Actions 自动部署到 GitHub Pages。

#### 首次设置

1. 在 GitHub 仓库设置中启用 Pages：
   - 进入仓库的 `Settings` → `Pages`
   - 在 `Source` 中选择 `GitHub Actions`

2. 推送代码到 `main` 分支，GitHub Actions 会自动构建并部署

3. 部署完成后，访问：`https://<username>.github.io/<repository-name>/`

#### 自动部署

- 每次推送到 `main` 分支时自动触发部署
- 也可以手动触发：在 Actions 标签页选择 workflow 并点击 `Run workflow`

#### 注意事项

- 如果仓库名为 `username.github.io`，应用会部署到根路径
- 如果仓库名为其他名称，应用会自动部署到 `/repository-name/` 子路径
- 构建过程会自动处理 base path 配置

## 游戏规则

掼蛋是一种流行于江苏及周边地区的扑克牌游戏：

- 使用两副牌（108张），4人游戏，2v2团队对战
- 从2开始升级，依次到A，2不必打，A必打，先过A的队伍获胜
- 支持多种牌型：单张、对子、三张、三带二、三连对、钢板、顺子（恰好5张）、同花顺、炸弹、四王
- 牌型大小：四王 > 六张及以上炸弹 > 同花顺 > 五张炸弹 > 四张炸弹 > 其它牌型
- 主牌大于副牌，红桃主牌为"逢人配"（可任意组合）
- 队友可以不出牌，让队友继续出
